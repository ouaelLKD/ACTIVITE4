Les TIC sont présentés dans 3 grandes catégories :
  • La téléinformatique: permettent l’échange d’informations numériques entre plusieurs systèmes informatiques.

Les objectifs sont d'atteindre la connectivité, diminuer l'ensemble d’appareils généraux, faciliter l'utilisation et l'implantation (solution générale aux problemes), détection et correction des erreurs.

1- Le réseau informatique :


Est un ensemble d'ordinateurs et de périphériques autonomes connectés entre eux et situés dans une zone géographique déterminée, qui échangent des informations numériques selon des règles définies.

La mise en réseau: (ou "Networking"), désigne la mise en œuvre d'outils et de tâches qui permettent la connexion d'ordinateurs afin de partager des ressources.

Les Réseaux informatiques sont nés du besoin de faire communiquer des terminaux distants avec un site central. Dans un premier temps ces communications étaient juste destinées aux transports de données mais maintenant on inclu la parole, et la vidéo. Un réseau s’appuie sur deux notions:L’interconnexion et La communication

2- Les caractéristiques de base d’un réseau :


La topologie, la topologie logique, le débit exprimé en bits/s (ou bps), la distance maximale (ou portée) et le nombre de noeuds maximum.

3-Terminologies en réseau :


  • Station de travail
  • Noeud
  • Paquet
  • Réseaux Hétérogènes
  • Le Débit

4- Éléments d'un reseau:


Les éléments d'un réseau sont:

  • Les ordinateurs
  • Les logiciels
  • Les supports du LAN
  • Les équipements d'interconnexion du LAN

5.1 Par portée l’envergure (La distance)


LAN (Local Area Network = réseau local d'entreprise)
un réseau informatique local qui relie des ordinateurs et des périphériques dans une zone géographique restreinte, telle qu'un bâtiment ou un campus.

MAN : (Métropolitan Area Network = Réseau métropolitain ou urbain) un réseau informatique qui couvre une zone géographique plus étendue, comme une ville ou une région métropolitaine.

WAN : (Wide Area Network = réseau grande distance) un réseau informatique qui couvre une grande zone géographique, comme un pays ou même plusieurs pays, et relie des ordinateurs et des réseaux locaux distants.

5.4 Le type d’organisation


Les réseaux Postes à Postes (Peer to Peer):
chaque périphérique est connecté directement à un ou plusieurs autres périphériques, sans passer par un point central, permettant une communication directe entre les périphériques.



Les réseaux Clients/Serveurs :
les périphériques, appelés clients, envoient des requêtes à un autre périphérique appelé serveur, qui fournit des services ou des ressources en réponse à ces requêtes.

6 -Le type de topologie :


Il existe differentes types de topologie:

Bus (ETHERNET, Token-Bus...) :
les périphériques sont connectés à un même câble commun, appelé bus, permettant la communication entre eux.

Etoile (téléphonie, STARLAN…):
chaque périphérique est connecté un commutateur central (hub) , permettant la communication entre eux via ce point central.

Anneau (FDDI, Token-Ring):
les périphériques sont connectés dans un cercle fermé, où chaque périphérique est connecté aux périphériques adjacentes, permettant la communication entre eux en boucle.
Maillée (FDDI, Token-Ring):
chaque périphérique est connecté à plusieurs autres périphériques, qui permettent la communication entre eux via plusieurs chemins possibles

Le modèle OSI:



1- Les composants matériels d’un réseau informatique :

Les cartes réseaux :
La carte réseau d’un ordinateur permet de se connecter à l’internet et au réseau local , elle est installée sur la carte mère et elle permet de créer un lien entre chaque ordinateur pour une communication et des échanges faciles entre eux.

Switch :
Ou commutateur réseau, il relie plusieurs appareils dans un réseau local (LAN) , et permet de la communication entre eux en utilisant des adresses (MAC) de l’appareil connecté et la stocker dans une table de correspondance dont il l’utilise pour envoyer les données à un appareil destinataire sur le réseau .

Hub :
Ou connecteur est utilisé pour connecter plusieurs appareils sur un réseau local (LAN), il reçoit des données envoyée par un appareil et les transmet à tous les autres appareils connectés.
2-Le modèle OSI : (open Systems Interconnections):
Est une norme de communication en réseau entre ordinateurs, et qui devise les communications en 7 couches suivantes :
  • la couche physique
  • la couche liaison de données
  • la couche réseau
  • la couche transport
  • la couche session
  • la couche présentation
  • la couche application
Le modèle OSI est utile pour comprendre les principes fondamentaux de la communication de réseau

3-le modèle TCP/IP : (Transmission Control Protocol / Internet Protocol)

C’est un modèle qui devise les communications en 4 couches principales :
  • couche liaison des données
  • couche réseau
  • couche transport
  • couche application
Il est utilisé pour la conception et la mise en œuvre des réseaux informatiques modernes.

4/2-Une adresse IP (Internet Protocol) : est le numéro qui identifie chaque ordinateur dans un réseau.

4/3-les modes d’adressage :
4/3/1- adresse IPV4 :
Chaque paquet source (4octets) et une adresse destinataire (8octets)
  • les 4 octets composent 2 parties : réseau et hôtes.
  • le nombres de bits contenus dans la partie hôte détermine le nombre de machines possible sur ce réseau
  • un réseau est hôte si la valeur des bits pour la partie réseau est la même.

4/3/2- les types d’adresse IPV4 :
  • L’adresse réseau
  • L’adresse de diffusion
  • L’adresse des hôtes
  • masque de réseau et de sus réseaux

4/3/3- L’adresse IPV6 : les adresses sont codées sur 128 bits

La gestion des informations selon Git :



Git pense les données à la manière d’un flux d’instantanés Chaque instantané est stocké dans une base de données localeLe fait que l’on dispose de l’historique complet d’un projet localement fait que la grande majorité des opérations de Git peuvent être réalisées localement


Les états des fichiers :



. Un fichier possède l’état “suivi” s’il appartenait au dernier instantané capturé par Git
. Lors du démarrage d’un dépôt Git à partir d’un dépôt local, on demande à Git de valider l’ensemble des fichiers du projet
. lorsqu’on va travailler sur notre projet, on va certainement ajouter de nouveaux fichiers ou modifier des fichiers existants
. lorsqu’on demande à Git de prendre l’instantané les fichiers faisant partie de l’instantané sont à nouveau considérés comme “validés” et le cycle peut recommencer.


Les zones de travail :



Tout projet Git est composé de trois sections :
. Le répertoire de travail correspond à une extraction unique d’une version du projet.
. La zone d’index correspond à un simple fichier.
. Le répertoire Git est l’endroit où Git stocke les méta-données et la base de données des objets de votre projet.

14- Ajouter ou modifier des fichiers dans un projet etactualiser notre dépôt Git:



Lorsque des changements sont effectués sur le projet, il est important d'indiquer ces modifications à Git en utilisant les commandes git add et git commit. Il est crucial de comprendre que la validation d'un fichier se fait en fonction de son état au moment du git add. Ainsi, pour enregistrer la dernière version d'un fichier, il est recommandé de faire un git add juste avant un git commit. Il est également possible d'utiliser l'option -a de git commit pour valider les fichiers modifiés et déjà suivis sans avoir à utiliser git add.


16- Annuler des modifications apportées à un fichier :



utiliser un logiciel de gestion de version. Après avoir enregistré un commit, il est possible de revenir à un état précédent du fichier en utilisant la commande git checkout ou git restore.


17- Qu’est ce qu’une branche ? :



la branche est un outil de développement de logiciel permettant de tester de nouvelles fonctionnalités sans affecter le projet de base. Git crée simplement un nouveau pointeur vers un commit en particulier.
